# CS 페이지 UI 스펙

## A. 고객용 페이지 (`/cs/*`)

### A-1. 인증 페이지 (`/cs`)

```
┌──────────────────────────────────────┐
│          1Day1Message                │
│          고객 문의                    │
│                                      │
│  ┌──────────────────────────────┐    │
│  │ 주문번호                     │    │
│  └──────────────────────────────┘    │
│  ┌──────────────────────────────┐    │
│  │ 전화번호 뒷 4자리            │    │
│  └──────────────────────────────┘    │
│                                      │
│  [ 조회하기 ]                        │
│                                      │
│  * 인증 실패 시 에러 메시지          │
│  * Rate limit 초과 시 안내           │
└──────────────────────────────────────┘
```

- 컴포넌트: `Card`, `Input`, `Button`
- 심플한 센터 레이아웃 (max-w-md)
- 로고 상단 표시

```
┌──────────────────────────────────────┐
│ 1D1M │ {고객명}님  [로그아웃]         │
├──────────────────────────────────────┤
│                                      │
│ ■ 구독 현황                          │

│ │ 상품명    │ Day │ D-Day │ 상태  │ │

│ │ 오늘의명언│ 15  │ D-350 │ 발송중│ │
│ │ 영어회화  │  3  │ D-362 │ 발송중│ │
│ └──────────────────────────────────┘ │
│                                      │
│ ■ 문의 내역                          │
│ ┌──────────────────────────────────┐ │
│ │ [메시지 미수신] 메시지가 안와요  │ │
│ │ 2025-04-02 │ 답변완료            │ │
│ ├──────────────────────────────────┤ │
│ │ [일시정지] 여행가서 정지 부탁    │ │
│ │ 2025-04-01 │ 처리완료            │ │
│ └──────────────────────────────────┘ │
│                                      │
│ [ + 새 문의 작성 ]                   │
└──────────────────────────────────────┘
```

- 컴포넌트: `Card`, `Table`, `StatusBadge`, `Button`, `EmptyState`
- 구독 현황: 간단한 테이블 (상품명, Day, D-Day, 상태)
- 문의 내역: 카드 리스트 (카테고리, 제목, 날짜, 상태)
- 문의 클릭 → 상세 페이지

### A-3. 문의 작성 (`/cs/dashboard` 내 FormDialog)

```
┌──────────────────────────────────────┐
│ 새 문의 작성                         │
├──────────────────────────────────────┤
│ 카테고리 [▼ 메시지 미수신       ]    │
│                                      │
│ 관련 구독 [▼ 오늘의 명언 (15일차)]   │
│   (선택사항, 복수 구독 시 표시)      │
│                                      │
│ 제목                                 │
│ ┌──────────────────────────────────┐ │
│ │                                  │ │
│ └──────────────────────────────────┘ │
│                                      │
│ 내용                                 │
│ ┌──────────────────────────────────┐ │
│ │                                  │ │
│ │                                  │ │
│ └──────────────────────────────────┘ │
│                                      │
│             [ 취소 ] [ 문의 등록 ]   │
└──────────────────────────────────────┘
```

- 컴포넌트: `FormDialog`, `Select`, `Input`, `Textarea`
- 카테고리 필수 선택
- 관련 구독 선택 (선택사항, 일시정지/상품변경 시 AI 참조용)
- 제출 → AI 자동 응답 → 결과 페이지로 이동

### A-4. 문의 상세 (`/cs/inquiry/[id]`)

```
┌──────────────────────────────────────┐
│ ← 목록으로                           │
├──────────────────────────────────────┤
│ [메시지 미수신] 메시지가 안와요      │
│ 2025-04-02 14:30 │ 답변완료          │
├──────────────────────────────────────┤
│                                      │
│ ┌─ 고객 ────────────────────────┐   │
│ │ 3일 전부터 메시지가 안 옵니다  │   │
│ └────────────────────────────────┘   │
│                                      │
│ ┌─ 담당자 ──────────────────────┐   │
│ │ 문의 주셔서 감사합니다.        │   │
│ │ 현재 오늘의 명언 구독 15일차...│   │
│ └────────────────────────────────┘   │
│                                      │
│ ┌─ 고객 ────────────────────────┐   │
│ │ 다 했는데 안 옵니다            │   │
│ └────────────────────────────────┘   │
│                                      │
│ ┌─ 담당자 ──────────────────────┐   │
│ │ 카카오톡 설정 확인 부탁드립... │   │
│ └────────────────────────────────┘   │
│                                      │
│ ■ 추가 문의                          │
│ ┌──────────────────────────────────┐ │
│ │                                  │ │
│ └──────────────────────────────────┘ │
│ [ 답변 등록 ]                        │
└──────────────────────────────────────┘
```

- 게시판 스타일: 원글 + 댓글 목록
- 댓글 구분: 고객 (왼쪽), 담당자/AI (오른쪽 또는 배경색 구분)
- 하단에 추가 답변 입력 (Textarea + Button)
- AI/관리자 답변은 "담당자"로 통일 표시 (고객에게 AI인지 구분 불필요)

## B. 관리자 CS 탭 (`Dashboard > CS`)

### B-1. 전체 레이아웃

```
┌──────────────────────────────────────────────┐
│ CS                                           │
├──────────────────────────────────────────────┤
│ [확인 필요 (3)] [AI 응대 (12)] [운영 정책]   │
├──────────────────────────────────────────────┤
│                                              │
│ (선택된 섹션 내용)                            │
│                                              │
└──────────────────────────────────────────────┘
```

- `PageHeader` + 서브탭 (3개 섹션)
- 카운트 뱃지 표시

### B-2. 확인 필요 섹션

```
┌──────────────────────────────────────────────┐
│ 고객명  │ 카테고리     │ 제목      │ 접수일  │
├─────────┼──────────────┼───────────┼─────────┤
│ 홍길동  │ 메시지 미수신│ 카톡ID:.. │ 04-02   │
│         │              │ [답변] [스킵]       │
├─────────┼──────────────┼───────────┼─────────┤
│ 김영희  │ 기타         │ 상담 요청 │ 04-01   │
│         │              │ [답변] [스킵]       │
└──────────────────────────────────────────────┘
```

- `DataTable` 사용
- 행 클릭 또는 답변 버튼 → `FormDialog`로 답변 작성
- 스킵 → `ConfirmDialog`로 확인

### B-3. AI 응대 섹션

```
┌──────────────────────────────────────────────┐
│ 고객명  │ 카테고리     │ 제목      │ 응대일  │
├─────────┼──────────────┼───────────┼─────────┤
│ 이철수  │ 일시정지     │ 정지 요청 │ 04-02   │
│         │ AI: 처리완료 │                      │
├─────────┼──────────────┼───────────┼─────────┤
│ 박민수  │ 메시지 미수신│ 안 옴     │ 04-02   │
│         │ AI: 온보딩안내│                     │
└──────────────────────────────────────────────┘
```

- `DataTable` 사용
- 행 클릭 → `DetailModal`로 대화 내역 확인
- 관리자가 추가 답변 가능

### B-4. 운영 정책 섹션

```
┌──────────────────────────────────────────────┐
│ 카테고리      │ 제목          │ 최종 수정    │
├───────────────┼───────────────┼──────────────┤
│ 메시지 미수신 │ 메시지 미수신 │ 04-01 14:30  │
│ 일시정지/재개 │ 일시정지/재개 │ 03-28 10:00  │
│ 상품 변경     │ 상품 변경     │ 03-28 10:00  │
│ 취소/환불     │ 취소/환불     │ 03-25 09:00  │
│ 발송 시간     │ 발송 시간     │ 03-20 16:00  │
│ 온보딩 절차   │ 온보딩 절차   │ 03-15 11:00  │
└──────────────────────────────────────────────┘

행 클릭 → 편집 다이얼로그:
┌──────────────────────────────────────┐
│ 운영 정책 수정 — 메시지 미수신       │
├──────────────────────────────────────┤
│ 정책 내용 (마크다운)                 │
│ ┌──────────────────────────────────┐ │
│ │ ## 메시지 미수신                 │ │
│ │ AI가 시스템으로 확인할 수 없...  │ │
│ └──────────────────────────────────┘ │
│                                      │
│ AI 지시사항                          │
│ ┌──────────────────────────────────┐ │
│ │ 1차 응답에서 반드시 3단계 안내.. │ │
│ └──────────────────────────────────┘ │
│                                      │
│             [ 취소 ] [ 저장 ]        │
└──────────────────────────────────────┘
```

- `DataTable` + `FormDialog`
- 마크다운 에디터 (Textarea로 충분)
- `content`: 고객에게 보여줄 정책 텍스트
- `ai_instruction`: AI에게 주는 지시사항 (관리자만 보임)
