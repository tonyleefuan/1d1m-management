# CLAUDE.md 슬림화 + 모듈 문서 분리

CLAUDE.md가 비대해지면 Claude Code 컨텍스트 윈도우를 잡아먹어 대화가 과도하게 압축됩니다.
아래 절차에 따라 CLAUDE.md를 슬림화하고, 상세 정보를 모듈별 문서로 분리하세요.

---

## 절차

### 1단계: 현황 분석
1. `wc -l CLAUDE.md`로 현재 줄 수 확인
2. CLAUDE.md를 전체 읽고 섹션별로 **유지/이동/삭제** 분류:
   - **유지**: 프로젝트 개요, 기술 스택, 실행 커맨드, 필수 규칙, 핵심 비즈니스 정의, 코드 규칙 요약, 디렉토리 구조
   - **이동**: 모듈별 파일 맵, 스키마 상세, API 라우트 목록, 스타일 가이드 상세, 코드 예시
   - **삭제**: docs/에 이미 존재하는 중복 내용, SQL 마이그레이션 목록 등

### 2단계: 모듈별 docs에 파일 맵 추가
1. 기존 docs/ 파일들을 읽어서 **파일 맵**(탭/API/Lib/DB) 정보가 있는지 확인
2. 없으면 각 문서 상단(제목+소개 바로 아래)에 표준 형식으로 추가:

```markdown
## 파일 맵

| 레이어 | 파일 |
|--------|------|
| **탭** | `컴포넌트명.tsx` (줄수) |
| **API** | `api/모듈/*` (N개 라우트) — 라우트1, 라우트2 |
| **Lib** | `파일명.ts` (줄수) |
| **DB** | `테이블1`, `테이블2`, `테이블3` |
| **관련 문서** | [문서명](상대경로) |
```

3. CLAUDE.md에만 있고 docs/에 없는 정보가 있으면 해당 docs로 이동

### 3단계: CLAUDE.md 재작성
목표: **100줄 이하**. 아래 구조를 따르세요:

```markdown
# 프로젝트명

한줄 소개.

## 기술 스택
- (5줄 이내)

## 실행/배포
- (커맨드만)

---

## 모듈별 문서 라우팅

> 작업할 모듈의 문서를 읽고 시작하세요. 각 문서 상단에 파일 맵이 있습니다.

| 모듈 | 상태 | 문서 |
|------|------|------|
| 모듈A | 상태 | [문서링크] |
| ... | ... | ... |

### 횡단 문서
| 문서 | 용도 |
|------|------|
| [스키마](docs/SCHEMA.md) | DB 스키마 SSOT |
| ... | ... |

---

## 필수 규칙
1. (번호 리스트, 각 1줄)

## 핵심 비즈니스 정의
- (CM1/CM2 등 코드 전반에 영향주는 정의만)

## 코드 규칙
- (한줄씩 간결하게, 코드 블록 최소화)

## 디렉토리 구조
(간략한 트리)
```

### 4단계: 검증
1. `wc -l CLAUDE.md` — 100줄 이하인지 확인
2. CLAUDE.md의 모든 마크다운 링크가 실제 파일을 가리키는지 확인
3. 모듈 docs 내부 링크도 검증
4. 삭제된 정보 중 docs/에도 없는 것이 있으면 복구

---

## 원칙

- CLAUDE.md는 **라우팅 테이블** 역할만 — "어떤 작업 → 어떤 문서 읽어라"
- 상세 정보(스키마, API, 파일 목록, 스타일 가이드)는 **모듈별 docs/**에
- Claude Code는 필요할 때 docs/를 읽을 수 있으므로, CLAUDE.md에 전부 넣을 필요 없음
- 정보 손실 없이 이동만 — 삭제가 아니라 재배치
